# UX Service Audit

Audit a complex service across touchpoints, roles, systems, approvals, and handoffs.

## Use When

- 问题不在单页，而在注册、审批、通知、handoff、支持等整条流程
- 体验跨多个角色、系统、渠道或线下线上交接
- 你想先找结构性断点，再决定哪些页面需要改

## Input

- `service_scope`：服务范围、关键触点和涉及系统
- `primary_roles`：主要用户、协作者、审批者或运营角色
- `critical_scenarios`：最重要的任务场景、失败点或投诉点

## Returns

- 服务链路断点清单
- 角色与交接问题判断
- 下一轮结构性修复方向

## Notes

- 适合服务设计、复杂工作流、AI agent handoff 和跨系统体验
- 如果你只想快速找一个页面哪里不清楚，先用 `ux-clarity-audit`
