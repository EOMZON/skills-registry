# Evidence Intake

Use this reference before critiquing layout, copy, or flow details.

## Goal

Anchor the audit in real friction signals instead of only taste or intuition.

## Useful sources

- analytics and funnel drop-off
- support tickets or chat questions
- usability notes
- search queries
- sales or onboarding feedback
- known complaints from stakeholders

## Questions

1. What evidence says users are confused or blocked?
2. Where in the path does that happen?
3. Which issue is frequent versus anecdotal?
4. What do we still have to infer?

## Output shape

```text
Evidence:
- ...

Known breakpoint:
- ...

Assumption still in play:
- ...
```
