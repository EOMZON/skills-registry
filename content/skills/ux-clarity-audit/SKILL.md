# UX Clarity Audit

Review a homepage, page, board, or key flow to find where people get confused, hesitate, or take the wrong path.

## Use When

- 首页说不清你是谁、做什么、适合谁
- 页面入口很多，但用户不知道先点哪个
- 你想快速定位信息架构、文案、CTA、状态和关键路径里的清晰度问题

## Input

- `target`：要审查的页面、网址、产品或关键流程
- `primary_goal`：用户最重要的任务或你最担心的关键路径
- `known_breakpoints`：已知困惑点、跳出点或重点模块

## Returns

- 优先级问题清单
- 首页或流程清晰度判断
- 三条最值得先改的动作

## Notes

- 默认不只看首屏，也会看用户从入口到结果的关键路径
- 如果问题跨多个角色、系统和 handoff，改用 `ux-service-audit`
