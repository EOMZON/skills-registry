# Minimalist Information Presentation

Applicable to any page, UI, document, dashboard, landing page, portfolio, or tool interface.

## What minimalism means

Minimalism is editorial discipline, not a visual style. The goal is not to look clean. The goal is to remove every obstacle between the viewer and the information they need.

A page is minimal when nothing can be removed without losing meaning.

## Fundamental question

Before placing any element, ask: what decision or action does this help the viewer take?

If there is no answer, the element does not belong on the page.

## Core laws

### 1. One signal at a time

Treat attention as a single beam. Make the page answer one question at a time:

- Who or what is this?
- Why does it matter to me?
- What should I do next?

Answer them in sequence, not simultaneously.

### 2. Hierarchy before aesthetics

Set the visual order before styling:

- Level 1: one primary message
- Level 2: one supporting message
- Level 3: entry points and actions
- Level 4: metadata and secondary detail

If two elements feel equally important, one of them is wrong.

### 3. One home per idea

Do not repeat an idea just to reassure the viewer. If it already exists on the page, keep the stronger version and delete the weaker one.

### 4. Explain less, show earlier

Lead with the thing itself. Projects beat project descriptions. Evidence beats explanation. Interface beats preamble.

### 5. Make every layer earn its place

Sections, dividers, tags, icons, labels, and motion all add cognitive load. Ask what breaks if you remove each one. If nothing breaks, remove it.

## Minimum viable sequence

### Orient

Answer what this is and why the viewer should care with one headline and one subline.

### Prove

Show the fewest, strongest proofs. Prefer entry points and real artifacts over summaries about summaries.

### Act

Present one clear next step once the proof has landed.

## Common failure patterns

### Density doubling

Show one version at a time instead of duplicating the same content across languages, formats, or phrasings.

### Reassurance loops

Delete sections that only defend or restate another section.

### Competing primaries

Demote everything until a single message and a single main action lead the page.

### Structural inflation

Merge overlapping sections instead of spreading one idea across many containers.

## Editing test

1. Summarize the page in one sentence.
2. Find every element that does not serve that sentence.
3. Remove it.
4. Repeat until further removal would damage the meaning.

Minimalism comes from subtraction, not addition.
