# Minimalist Information Presentation

## Overview

Treat minimalism as editorial discipline, not a visual style. Remove friction between the viewer and the decision they need to make.

## Workflow

1. Define the page job
   - Write one sentence for what the page must help the viewer decide or do.
   - If an element does not support that sentence, plan to remove or demote it.

2. Reduce the page to one sequence
   - `Orient`: answer what this is and why it matters.
   - `Prove`: show the smallest set of evidence, examples, or entry points that make the claim credible.
   - `Act`: present one clear next step.
   - Force every element into one of these stages. Cut anything that fits none of them.

3. Enforce the core laws
   - One signal at a time: let the page answer one viewer question per moment, not several at once.
   - Hierarchy before aesthetics: define one primary message, one supporting message, one action layer, then everything else.
   - One home per idea: if the same idea appears twice, keep the stronger instance and delete the weaker one.
   - Show earlier, explain less: lead with the actual work, evidence, or interface instead of describing what is coming.
   - Make every layer earn its place: remove sections, labels, dividers, tags, icons, and motion unless something breaks without them.

4. Run the subtraction pass
   - Demote or remove anything competing with the primary message.
   - Merge sections that say the same thing in different wrappers.
   - Replace reassurance copy with stronger evidence.
   - Prefer direct examples, projects, outputs, or actions over meta-description.

5. Check the action path
   - Keep one primary action per page state unless the product genuinely requires multiple equal paths.
   - Make the action visible after the proof is established, not buried in lower-priority blocks.

## Failure Patterns

- Density doubling: showing the same content in two phrasings, formats, or languages at once.
- Reassurance loops: adding sections whose only job is to explain why another section matters.
- Competing primaries: multiple headlines, buttons, or modules fighting for the same weight.
- Structural inflation: splitting one idea across `About`, `Approach`, `Philosophy`, and `Values` when one strong section would do.

## Editing Test

1. Cover the page and summarize it in one sentence.
2. Uncover it and mark every element that does not serve that sentence.
3. Remove the marked elements.
4. Repeat until removing anything meaningful would weaken the message.
